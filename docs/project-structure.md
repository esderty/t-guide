# Структура проекта

Этот файл описывает назначение основных файлов проекта `t-guide` и объясняет, как сейчас работает приложение.

## 1. Корень проекта

| Файл | Назначение |
| --- | --- |
| `README.md` | Главная документация проекта: описание, запуск, стек, основные сценарии. |
| `.gitignore` | Список файлов и директорий, которые не должны попадать в git. |
| `.env.example` | Пример переменных окружения для карты и backend. |
| `.env.local` | Локальные переменные окружения разработчика. В репозиторий не коммитится. |
| `package.json` | Скрипты, зависимости и devDependencies проекта. |
| `package-lock.json` | Зафиксированное дерево зависимостей `npm`. |
| `vite.config.ts` | Конфигурация Vite, alias `@`, настройки `localhost`. |
| `tsconfig.json` | Базовая конфигурация TypeScript. |
| `tsconfig.app.json` | TypeScript-конфиг для клиентского приложения. |
| `tsconfig.node.json` | TypeScript-конфиг для Node/Vite-конфигурации. |
| `eslint.config.js` | Настройка ESLint для проекта. |
| `index.html` | Корневой HTML-файл Vite, в который монтируется React. |

## 5. Слой `app`

### `src/main.tsx`

Точка входа в приложение. Подключает глобальные стили и монтирует `App` в DOM.

### `src/app/App.tsx`

Корневой компонент приложения.

Отвечает за:

- подключение `BrowserRouter`;
- общий layout;
- липкий `header`;
- навигацию между главной и каталогом экскурсий;
- рендеринг `AppRouter`.

### `src/app/providers/AppRouter.tsx`

Описывает маршруты React Router:

- `/`
- `/excursions`
- `/excursions/:slug`
- `404`

### `src/app/styles/index.css`

Глобальные стили всего приложения.

Здесь описаны:

- layout;
- header;
- карточки;
- фильтры;
- карта;
- список точек;
- аудиопанель;
- адаптивное поведение на мобильных экранах.

## 6. Слой `pages`

### `src/pages/home/ui/HomePage.tsx`

Главная страница приложения.

Показывает:

- название и описание приложения;
- общую статистику по экскурсиям;
- последние 3 экскурсии;
- кнопку перехода ко всему каталогу.

### `src/pages/excursions/ui/ExcursionsPage.tsx`

Страница каталога экскурсий.

Показывает:

- баннер каталога;
- счетчики по всем и отфильтрованным маршрутам;
- поиск;
- фильтры;
- список карточек экскурсий;
- пагинацию по 6 карточек.

Также эта страница:

- использует `useDeferredValue` для более плавной фильтрации;
- использует `runViewTransition` для мягких переходов;
- удерживает минимальную высоту каталога, чтобы экран не дергался.

### `src/pages/excursion/ui/ExcursionPage.tsx`

Страница одной экскурсии.

Отвечает за:

- загрузку маршрута по `slug`;
- хранение активной точки маршрута;
- вывод метаинформации о маршруте;
- сборку блока `RouteOverview`;
- вывод карточки места и аудиопанели.

### `src/pages/not-found/ui/NotFoundPage.tsx`

Страница 404 для несуществующих маршрутов и URL.

### `src/features/place-details/ui/PlaceDetailsCard.tsx`

Карточка выбранной точки маршрута.

Показывает:

- изображение;

- краткое описание;
- номер точки;
- координаты;

### `src/features/audio-guide/ui/AudioGuidePanel.tsx`

Панель аудиогида по выбранной точке.

Содержит:

- краткий превью-текст аудио;
- длительность;

Сейчас MP3-файлы не подключены, поэтому чаще всего отображается заглушка.

### `src/features/route-map/ui/RouteMap.tsx`

Точка переключения провайдера карты.

Если включен `yandex` и есть ключ, используется `YandexRouteMap`.
Если нет, используется `RouteMapPreview`.

### `src/features/route-map/ui/YandexRouteMap.tsx`

Главный интерактивный компонент карты.

Отвечает за:

- загрузку и инициализацию Яндекс Карт;
- отрисовку точек маршрута;
- отображение стартовой и конечной точки;
- отображение пользователя по геолокации;
- переключение между точками;
- кнопки управления под картой;
- построение bounds для всего маршрута;
- fallback-режим при недоступности карты.

Сейчас логика такая:

- карта отображается по `VITE_YANDEX_MAPS_API_KEY`;
- роутинг пытается строиться через `VITE_YANDEX_ROUTER_API_KEY`;
- если router API не отвечает, маршрут падает в запасной режим по прямым линиям.

### `src/features/route-map/ui/RouteMapPreview.tsx`

Упрощенная карта-заглушка без внешнего API.

Используется, когда:

- провайдер карты выключен;
- нет ключа;
- интерактивная карта не загрузилась.

### `src/features/route-map/model/types.ts`

Типы для карты:

- `MapProvider`;
- `RouteMapProps`.

### `src/features/route-map/model/useUserGeolocation.ts`

Хук геолокации пользователя.

Отвечает за:

- `watchPosition`;
- определение текущей позиции;
- повторный запрос геолокации;
- статусы `loading / tracking / blocked / unsupported`;
- тексты ошибок по геолокации.

### `src/features/route-map/lib/route-geometry.ts`

Набор утилит для построения геометрии маршрута.

Содержит:

- преобразование координат в формат Яндекса;
- вычисление bounds;
- построение fallback-маршрута;
- попытку построить пешеходный маршрут через router API;
- сборку маршрута из сегментов `точка -> следующая точка`;
- поиск ближайшей точки к пользователю;
- форматирование расстояния.

### `src/features/route-map/lib/yandex-loader.ts`

Низкоуровневый загрузчик Яндекс Карт.

Отвечает за:

- динамическую загрузку JS API;
- пробу `api-maps` и `enterprise.api-maps`;
- инициализацию `@yandex/ymaps3-reactify`;
- установку router key через `setApikeys({ router })`;
- кеширование загруженного API;
- сброс загрузчика при retry.

## 11. Как сейчас работает приложение

### Главная страница

1. `HomePage` вызывает `useExcursions()`.
2. Хук получает список маршрутов из репозитория.
3. Репозиторий либо берет моки, либо делает HTTP-запрос.
4. Главная показывает общую статистику и последние 3 экскурсии.

### Каталог экскурсий

1. `ExcursionsPage` получает список экскурсий.
2. Фильтры и поиск применяются через `filterExcursions`.
3. Результат разбивается на страницы по 6 карточек.
4. Каталог отрисовывается через `ExcursionCatalog`.

### Страница экскурсии

1. `ExcursionPage` читает `slug` из URL.
2. Репозиторий возвращает нужную экскурсию.
3. Страница выставляет первую точку как активную.
4. `RouteOverview` отображает карту и список точек.
5. Справа показываются карточка выбранного места и аудиопанель.

### Карта

1. `RouteMap` выбирает реальную карту или preview.
2. `YandexRouteMap` грузит API Яндекса.
3. Если карта доступна:
   - рисуются маркеры;
   - рисуется маршрут;
   - показывается геопозиция пользователя;
   - доступны кнопки управления.
4. Если карта или router API не сработали:
   - карта уходит в fallback-режим;
   - маршрут может показываться как запасная схема.

## 12. Что в проекте временное

Сейчас временными считаются:

- моковые экскурсии;

- fallback-карта;
- запасные прямые линии для маршрута, если router API Яндекса не отвечает.
